# SDING AI 플래너 — 베타 프로토타입 기획서 (v2)

> **이 문서의 용도**: Claude Code로 프론트엔드 프로토타입을 개발하기 위한 기획서입니다. 실제 AI 응답, 실제 결제, 실제 업체 데이터베이스, 실제 사진 업로드/저장 등 **기능 구현은 범위에 포함되지 않습니다.** 모든 데이터는 Mock(가짜) 데이터로 처리하고, UI/UX 흐름이 실제처럼 "그럴듯하게" 동작하는 것이 목표입니다.
>
> **v2 변경사항**: 진입 화면을 채팅 중심 → **대시보드 중심**으로 전면 재설계했습니다. 채팅방은 대시보드에서 별도로 진입하는 화면이 되었고, 체크리스트는 대시보드에서 바로 접근할 수 있습니다. 업체 카테고리를 13종 → 14종으로 확장하고, 체크리스트를 31개 항목으로 대폭 보강했습니다.

---

## 1. 개요

### 1.1 배경
SDING 앱은 홈 / 전체서비스 / **AI플래너** / 커뮤니티 / 마이페이지 5개 탭으로 구성됩니다. 이번 베타는 이 중 AI플래너 탭 하나만 독립적으로 떼어내 먼저 프로토타입으로 만들면서 기획을 구체화하는 것이 목적입니다.

### 1.2 이 베타에서 만들 것
1. **대시보드** — AI플래너 탭 진입 시 처음 보이는 화면. 결혼 준비 진행률을 한눈에 보여주고, AI 드레스합성·AI 웨딩홀합성 배너, 체크리스트 미리보기, AI 채팅방 진입 카드를 포함.
2. **AI 웨딩플래너 채팅방** — 대시보드에서 별도로 진입하는 대화형 화면. 등록 업체를 추천하고 웨딩 관련 정보를 안내.
3. **웨딩 체크리스트** — 대시보드에서 진입. 31개 준비 항목을 그룹별로 보여주고, 업체 계약이 필요한 항목은 업체를 등록(사진 업로드 포함)하거나, 업체를 보러 가거나, 업체 등록 없이 바로 완료 처리할 수 있음.
4. **AI 드레스 합성** — 사용자 사진에 드레스를 가상으로 입혀보는 화면.
5. **AI 웨딩홀 합성** — 사용자(커플) 사진을 웨딩홀 배경에 합성해보는 화면.

### 1.3 이 베타에서 만들지 않는 것 (Out of scope)
- 실제 LLM API 연동 (AI 응답은 mock 시나리오 기반 스크립트로 처리)
- 실제 이미지 합성 AI (드레스/웨딩홀 결과는 미리 준비된 mock 이미지로 대체)
- 실제 파일 업로드/서버 저장 (사진 첨부는 브라우저 로컬 미리보기로 대체 — 6.3 참고)
- 로그인/회원가입, 실제 결제, 실제 업체 DB
- 홈 / 전체서비스 / 커뮤니티 / 마이페이지 탭 (다른 4개 탭은 이번 범위 아님. 최하단 5탭 네비게이션 바는 시각적으로 존재하되 AI플래너 외 탭은 비활성 처리 가능)

## 2. 기술 스택 제안

| 항목 | 제안 |
|---|---|
| 프레임워크 | React + Vite |
| 스타일링 | Tailwind CSS |
| 상태관리 | React useState/useContext (전역 상태: 체크리스트, 채팅 히스토리, 등록 업체) |
| 아이콘 | lucide-react |
| 애니메이션 | framer-motion (바텀시트, 배너 캐러셀, 프로그레스 애니메이션 등에 권장. 필수는 아님) |

## 3. 정보 구조 (IA) — v2

```
AI플래너 (진입점)
│
└── 1) 대시보드 (기본 화면)
      │
      ├── 진행률 요약 카드 (전체 진행률 + 그룹별 미니 진행률)
      │
      ├── AI 배너 캐러셀
      │     ├── "AI 드레스 합성" 배너 → 4) AI 드레스 합성 화면
      │     └── "AI 웨딩홀 합성" 배너 → 5) AI 웨딩홀 합성 화면
      │
      ├── 체크리스트 미리보기 ("다음 할 일" 2~3개 + 전체보기 버튼)
      │     └── 전체보기 → 2) 웨딩 체크리스트 화면
      │
      └── "AI 플래너에게 물어보기" 카드 → 3) AI 웨딩플래너 채팅방
      
2) 웨딩 체크리스트 화면 (대시보드에서 진입)
      └── 카테고리 연결 항목 탭 → 업체 처리 바텀시트
            ├── "업체 등록하기" → 업체 등록 폼 (사진 업로드 2장)
            ├── "업체 보러가기" → 해당 카테고리 업체 리스트 (mock, 전체서비스 탭 성격)
            └── "업체 등록 없이 완료" → 즉시 체크 완료 처리

3) AI 웨딩플래너 채팅방
      ├── 리치 카드(업체 추천, 체크리스트 넛지) 포함 대화
      ├── 우측 상단 더보기(⋮) → 간단 메뉴 (체크리스트 바로가기 / AI합성 바로가기 / 대시보드로)
      └── 뒤로가기 → 1) 대시보드로 복귀

4) AI 드레스 합성 화면
      └── "채팅으로 보내기" → 3) 채팅방으로 이동, 결과가 카드로 삽입
      └── 뒤로가기 → 1) 대시보드로 복귀

5) AI 웨딩홀 합성 화면 (4번과 동일 구조)
```

**v1 대비 핵심 변화**: 이전에는 채팅방이 기본 화면이고 체크리스트가 더보기 메뉴 안에 숨어 있었습니다. v2는 **대시보드가 기본 화면**이고, 체크리스트는 대시보드에서 바로 보이며, 채팅방은 "필요할 때 들어가는" 별도 공간이 되었습니다. 채팅방 내 더보기 메뉴는 여전히 존재하지만 이제는 "대시보드로 안 나가고 빠르게 이동하는 단축 경로" 역할로 축소되었습니다.

## 4. 핵심 UX 컨셉

- **대시보드 = 홈베이스.** "내 결혼 준비가 지금 어디까지 왔지?"를 3초 안에 파악할 수 있어야 합니다. 진행률, 다음 할 일, AI 기능이 모두 스크롤 없이(또는 한 번의 스크롤로) 보이는 것을 목표로 합니다.
- **채팅방 = 목적이 있을 때 들어가는 공간.** "AI한테 물어볼 게 있을 때" 들어가는 곳이라는 느낌을 주기 위해, 대시보드에 항상 눈에 띄는 카드/버튼("AI 플래너에게 물어보기")으로 진입 지점을 명확히 둡니다.
- **체크리스트 항목은 두 종류로 나뉩니다**: ① 업체 계약이 필요한 항목(카테고리 연결, 3가지 처리 방법 제공) ② 단순 준비 작업 항목(체크박스만 있음). 이 구분을 사용자가 직관적으로 알 수 있도록 아이콘이나 뱃지로 구분해주세요.

## 5. 화면별 상세 명세

### 5.0 대시보드 (신규, 기본 화면)

**레이아웃 (위→아래)**
1. 헤더: SDING 로고 + "OO님의 웨딩 플래너" · 우측에 알림 아이콘(선택)
2. **D-day 배너**: "결혼까지 D-186일" 같은 큰 텍스트 (mock 예식일 기준 계산)
3. **진행률 요약 카드**: 도넛/링 차트로 전체 진행률(예: "31개 중 13개 완료 · 42%") + 아래 그룹별 미니 progress bar 4~5줄(업체 계약, 청첩장, 게스트 준비, 본식 콘텐츠 등 — 5.2의 그룹 기준)
4. **AI 배너 캐러셀**: 가로 스크롤 또는 2열 배치. "AI 드레스 합성 — 내 사진에 드레스를 입혀보세요" / "AI 웨딩홀 합성 — 우리 부부가 이 홀에 선다면?" 각각 썸네일 이미지 포함한 큼직한 카드
5. **다음 할 일 미리보기**: 마감이 가까운/우선순위 높은 미완료 항목 2~3개를 카드로 표시 (제목 + D-day + 상태뱃지) + "체크리스트 전체보기 →" 버튼
6. **AI 플래너에게 물어보기 카드**: 하단에 고정성 카드 또는 플로팅 버튼. "궁금한 게 있으신가요? AI 플래너와 대화해보세요" + 챗 아이콘

### 5.1 AI 웨딩플래너 채팅방

v1 문서의 5.1과 동일합니다 (메시지 타입, mock 시나리오 A/B/C 그대로 유지). 차이는 진입/이탈 경로뿐입니다:
- **진입**: 대시보드의 "AI 플래너에게 물어보기" 카드에서 진입
- **이탈**: 헤더 뒤로가기 → 대시보드로 복귀 (v1에서는 이 화면이 기본 화면이라 뒤로가기 개념이 약했지만, v2에서는 명확히 "대시보드로 돌아간다"는 동작이 됨)
- **더보기(⋮) 메뉴 항목 축소**: 웨딩 체크리스트 바로가기 / AI 드레스합성 바로가기 / AI 웨딩홀합성 바로가기 / 대시보드로 가기 — 대시보드에 이미 다 있는 기능들의 "채팅 중 이탈 없이 빠르게 가는 단축 경로"로 성격이 바뀝니다.

### 5.2 웨딩 체크리스트 화면

**레이아웃**
1. 헤더: 뒤로가기(대시보드로 복귀) · "웨딩 체크리스트" 타이틀 · 진행률 텍스트("13/31")
2. 그룹별 아코디언 또는 섹션 리스트 (아래 표 기준, 8개 그룹)
3. 각 항목 행: 체크 상태 아이콘 + 항목명 + (카테고리 연결 항목인 경우) 카테고리 뱃지 + 상태 뱃지(미등록/상담중/계약완료/완료) + chevron

**전체 체크리스트 항목 (31개, 8개 그룹)**

- 부케순이 섭외

- **그 외 그룹(단순 체크형)** 탭 → 체크 토글 + 선택적으로 한 줄 메모 입력 필드가 펼쳐짐 (예: "예상 하객 리스트 작성" 탭 → 체크 + "메모: 약 150명 예상" 같은 짧은 텍스트 입력 가능. 필수 아님)

### 5.3 업체 처리 바텀시트 (신규 — 3가지 액션)

체크리스트 B그룹(업체 계약) 항목을 탭하면 뜨는 바텀시트. **세 가지 선택지를 명확히 병렬로 제시**합니다.

```
┌─────────────────────────────┐
│  웨딩홀 계약                  │
│  ───────────────────────    │
│  🏢  업체 등록하기            │
│      상품·계약서 사진을 올려요  │
│  ───────────────────────    │
│  🔍  업체 보러가기             │
│      스딩 등록 업체를 둘러봐요  │
│  ───────────────────────    │
│  ✓   업체 등록 없이 완료 처리   │
└─────────────────────────────┘
```

**옵션 1 — 업체 등록하기 (직접 입력 + 사진 업로드)**
탭하면 아래 필드를 가진 폼 화면(또는 확장된 바텀시트)이 열립니다.
- 상태 선택 (세그먼트: `상담중` / `계약완료`)
- 업체명 (텍스트 입력)
- **상품 사진 업로드** — 카메라 아이콘 버튼, 탭하면 `<input type="file" accept="image/*" capture="environment">` 로 사진 선택/촬영 → 로컬 미리보기 썸네일 표시 (실 업로드/서버 저장 없음, `URL.createObjectURL` 로 프리뷰만)
- **계약서 사진 업로드** — 위와 동일한 방식, 별도 슬롯
- 메모 (선택, 텍스트)
- 저장 버튼 → 체크리스트로 복귀, 항목에 상태뱃지 표시 + 계약완료인 경우 자동 체크

**옵션 2 — 업체 보러가기**
탭하면 해당 카테고리로 필터링된 mock 업체 리스트 화면으로 이동 (기존에 만든 `VendorSummary` mock 데이터 재사용, 5.1과 동일한 카드 형태). 이 화면은 "전체서비스" 탭의 카테고리 리스트 화면을 간이로 재현한 것으로 취급해도 됩니다 — 지금 베타 범위에서는 리스트만 보여줘도 충분하고, 실제 상세 페이지/예약까지는 만들지 않아도 됩니다. 리스트에서 업체를 하나 선택하면 "옵션 1"의 폼으로 자동 진입하며 업체명이 미리 채워집니다.

**옵션 3 — 업체 등록 없이 완료 처리**
탭 즉시 확인 토스트("웨딩홀 계약을 완료로 표시했어요") + 바텀시트 닫힘 + 체크리스트 항목이 업체 정보 없이 `완료` 상태로 바뀜 (상태뱃지는 "완료(업체 미등록)" 정도로 표시).

### 5.4 AI 드레스 합성 화면 / 5.5 AI 웨딩홀 합성 화면

v1 문서의 5.5/5.6과 동일합니다 (변경 없음). `SDING_입점제휴_제안서.docx`의 AI 드레스 피팅 / AI 웨딩홀 미리보기 목업을 기준으로 구현하고, 결과 화면에서 "채팅으로 보내기"를 누르면 대시보드가 아니라 **채팅방**으로 이동해 결과 카드가 삽입됩니다.

### 6.3 사진 업로드 mock 처리 방법

실제 서버 업로드 없이 "그럴듯하게" 보이려면:
```ts
const [preview, setPreview] = useState<string | null>(null);

function handleFileSelect(e: React.ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0];
  if (file) setPreview(URL.createObjectURL(file)); // 로컬 미리보기만, 저장/전송 없음
}
```
`<input type="file" accept="image/*" capture="environment">` 를 쓰면 모바일 브라우저에서 카메라 촬영도 바로 지원됩니다. 새로고침하면 사라져도 무방합니다(베타 목적상 세션 내 유지만 되면 충분).

## 9. 개발 단계 제안 (Claude Code 작업 순서)

1. **Phase 1 — 대시보드 셸**: 진행률 카드(더미 숫자), AI 배너 2개, 다음 할 일 미리보기, 채팅 진입 카드. 정적 레이아웃 먼저.
2. **Phase 2 — 체크리스트 화면**: 31개 mock 항목을 8개 그룹으로 렌더링, 체크 토글, 진행률 계산 로직(대시보드와 연동).
3. **Phase 3 — 업체 처리 바텀시트**: 3가지 옵션 UI, 업체 등록 폼(사진 업로드 mock 포함), 업체 보러가기 mock 리스트.
4. **Phase 4 — 채팅방**: v1 스펙 그대로 구현 (메시지 타입, mock 시나리오 3개, 축소된 더보기 메뉴).
5. **Phase 5 — AI 드레스/웨딩홀 합성 화면**: mock 이미지 선택 → 결과 표시 → 채팅 삽입.
6. **Phase 6 — 폴리싱**: 전환 애니메이션, 빈 상태 디자인, 반응형 점검, 대시보드 진행률 실시간 반영 확인.

## 10. 참고 및 제약사항

- 실제 AI 응답 대신 **키워드 매칭 기반의 간단한 mock 응답 엔진**을 권장합니다. 완전 자유 대화는 이번 베타 범위 밖입니다.
- 이미지 합성 결과는 실제 생성 없이 사전에 준비된 정적 이미지 3~5장을 순환 노출하는 방식으로 충분합니다.
- 사진 업로드는 6.3의 방식대로 로컬 미리보기만 구현하면 됩니다. 실제 저장/전송은 이번 범위가 아닙니다.
- **카테고리 14종 체계**를 이 문서 전체에서 사용했습니다. 기존 문서(업체 데이터수집 양식, B2B 파트너십 계약서 별첨1)는 13종 기준으로 만들어져 있어 용어가 어긋납니다 — 실제 통합 전에 하나로 맞춰야 합니다.
- 화면 간 전환은 "채팅방/체크리스트/AI합성 화면에서 나가면 대시보드로 돌아온다"는 원칙을 지켜주세요 (3장의 IA 다이어그램 참고).
